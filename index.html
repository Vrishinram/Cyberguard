Learn About Password Security

Understanding the fundamentals of cryptography and password security.

Password Entropy

Entropy measures the unpredictability of a password, expressed in bits. A password with N possible characters and length L has entropy = L × log₂(N). Higher entropy means exponentially more combinations an attacker must try.

Example 8-char lowercase: 8 × log₂(26) ≈ 37.6 bits
12-char mixed: 12 × log₂(94) ≈ 78.7 bits

Hashing & Salting

Passwords should never be stored in plaintext. Cryptographic hash functions (SHA-256, bcrypt, Argon2) create a fixed-size "fingerprint" of your password. Salting adds random data before hashing, preventing rainbow table attacks.

Best Practice Use Argon2id or bcrypt with a unique salt per user for password storage.

Common Attacks

Brute Force: Trying every combination.
Dictionary: Using lists of common passwords.
Credential Stuffing: Using leaked credentials across sites.
Phishing: Tricking users into revealing passwords.

Defense Use unique, high-entropy passwords with a password manager, and enable MFA.

Best Practices

• Use 12+ characters with mixed character types
• Never reuse passwords across services
• Use a reputable password manager
• Enable multi-factor authentication (MFA)
• Regularly check for breaches at haveibeenpwned.com

Tip Passphrases like correct-horse-battery-staple are both strong and memorable.
